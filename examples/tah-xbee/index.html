Tah to XBee Communications

Tah is a Bluetooth Low Energy open source development platform compatible with the Arduino.

Tah has onboard BLE (Bluetooth 4.0) and can be used with XBee networks to increase the range from 100m to 300m. Tah can be used as a bridge between Smart phone and XBee network.

With this feature of Tah the user can send commands and receives data over the XBee network and control and develope any wireless sensor network and access it with smart phone.

Requirements

  • Two Tahs
  • Two UNO Breakout Shields
  • Two S1/S2 XBee modules with Shields
  • Connecting wires
  • 12VDC power supply adapter or 9V HW Battery
  • PC/Laptop with Arduino IDE installed
  • Smartphone with Tah Write

This tutorial is divided into two major parts first one Tah transmitter section and other is receiver on the Robot which will control the motions of robot.


Step 1

Let's configure the XBee radios. Using a serial connection, program the two XBee radios to talk to each other at 9600 baud. You can use a program like HyperTerm or CoolTerm in Windows. Similarly for Linux you can use Minicom to configure the XBee radios.

This demo uses the default PAN ID of 3332, but you should choose a different one so that other XBee radios in the area do not interfere with your communications. For the first XBee, use the +++ sequence to go into command mode. You should receive an OK message back. Then issue this command:

ATID3332,ATMY10,ATDL11,ATBD3,ATWR,ATCN

Now you have configured the first XBee radio as PANID 3332, ATMY10 (16-bit source address), ATDL11 (Destination Low Address), ATBD3(9600 baud rate) and to confirm settings use ATWR and exit from command mode use ATCN.

Step 2

Now configure the second XBee. Connect your second XBee and again use the +++ sequence to go into command mode. Then issue this command:

ATID3332,ATMY11,ATDL10,ATBD3,ATWR,ATCN

The only difference here is that for the second XBee, you set the destination address to 10 and the source address to 11. The two XBees are now configured to talk to each other.


Step 3

In this demo we have implemented the Duplex communication in two XBee radios and sending data or commands from Smart phone using Tah Write app.

We are controlling onboard LEDs of Tah used in XBee network using smartphone. Here Tah has onboard BLE so Tah is acting as a communication bridge between Xbee and Smart phone.

In this demo command 1 is send from smart phone to Tah and Tah communicate to XBee and XBee coordinator wil send that data to End device of XBee, similarly End device will again send acknowledge signal to coordinator and LEDs will blink and vice versa.

Step 4

We are using Software Serial for XBee connections on Tah pin 8 is Rx and pin 9 is Tx of XBee. Connect Rx of XBee to Tx of Software Serial and Tx of XBee to Rx of Software Serial.

Download and install Tah arduino Library.

Download XBee coordinator Arduino Sketch and upload this sketch to Tah acting as coordinator

Download XBee as Enddevice and upload this sketch to Tah acting as end device

Plugin the both the Tahs and XBee shields on UNO Breakout shields.

Step 5

  • Now give the power supply to both the UNO Breakout Shields and open the Tah Write app
  • Scan for Tah Bluetooth Devices in the Tah Write app.
  • Connect with the device named TahXbee
  • Once connected entered 1 as command to make remote LED ON and 0 to make it OFF from phone.

    • You will also receive a notification from remote XBee on your phone.
